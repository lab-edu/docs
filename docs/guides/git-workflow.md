# Git 工作流

这一页定义团队日常开发时最基本的协作方式。

## 分支约定

- `main` 作为稳定分支
- `dev` 作为日常开发主分支
- 功能开发使用 `feature/*`
- 修复问题使用 `fix/*`

## 合并规则

- 所有变更必须通过 Pull Request 合并
- 不允许直接向 `main` 推送业务改动
- 合并前需要至少一人 Review

## 提交规范

- 提交信息优先使用结构化前缀，例如 `feat:`、`fix:`、`docs:`、`refactor:`
- 每次提交都要能说明实际改动
- 不使用含糊的提交描述

## 评审重点

- 接口是否一致
- 逻辑是否正确
- 是否破坏已有行为
- 文档是否同步更新
