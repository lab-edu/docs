# 通信方式

这一页说明各模块之间如何通信，以及哪些约定必须先统一。

## 当前约定

- web 通过同源入口访问 core 暴露的接口
- core 对外使用统一的 HTTP API
- api 路径统一使用 `/api/v1` 前缀
- 返回结构统一为 `code`、`message`、`data`

## 预留原则

- core 与 ai-service、fpga-service 之间先按“接口通信”处理
- 具体协议先不在 Phase 0 里锁死
- 如果后续业务需要更高吞吐或更强约束，再单独评估 gRPC 或其他方案

## 设计目标

- 降低模块间耦合
- 让前后端和预留服务都能独立演进
- 让接口文档先于实现存在，减少联调摩擦
